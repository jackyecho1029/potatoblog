### 观点一：从“画图”转向“构建”，AI 正在消除设计与工程的鸿沟

在 Notion 的设计流程中，Claude Code 不仅仅是一个对话框，而是一个能直接在本地代码库中操作的“数字工匠”。它让设计师能够跳过传统的静态模型，直接生成可运行的高保真原型。

**因為：**
- 静态原型（如 Figma）无法完全模拟复杂的交互逻辑和真实数据流动。
- AI Agent 具备上下文感知能力，能理解现有项目的代码结构并直接进行增删改查。

**案例/证据：** Notion 设计师 Brian Lovin 演示了如何通过简单的指令，让 Claude Code 在几秒钟内为一个 React 组件添加复杂的动态侧边栏功能。

## 📚 关键词

**1. 高保真原型（High-Fidelity Prototype）**

> **含义：** 极其接近最终产品的原型，包含真实的交互逻辑、动画和数据处理，而不仅仅是视觉上的模拟。

**💼 案例：** Airbnb 在早期通过编写实际运行的代码原型来测试其预订流程，这比单纯看静态设计图更能发现流程中的摩擦点，从而大幅提升转化率。

---

**2. AI 代理（AI Agent / Claude Code）**

> **含义：** 不同于简单的聊天机器人，AI Agent 能够理解任务目标，自主访问文件系统、执行终端命令并对代码库进行多步骤的复杂操作。

**💼 案例：** 许多硅谷初创公司开始使用此类工具让非技术人员（如电邮营销负责人或运营）直接修改产品的文案和简单逻辑，极大地减轻了工程部门的负担。

---

**3. 上下文感知（Context Awareness）**

> **含义：** AI 能够读取并理解整个项目目录、依赖关系和编码风格，从而提供精准、不冲突的代码建议。

**💼 案例：** Notion 的设计系统非常复杂，Claude Code 通过扫描整个代码库，能确保新生成的组件完美适配现有的 CSS 变量和组件规范，无需设计师手动纠错。

---

**4. 命令行界面工具（CLI Tool）**

> **含义：** 在终端运行的软件工具。对于 Claude Code 而言，这让它能直接与开发环境深度集成，实现比网页端 AI 更高的操作权限。

**💼 案例：** 高级开发者通常偏爱 CLI 工具。Stripe 提供了业界顶尖的 CLI，让开发者能像 Notion 设计师操作 AI 一样，在终端快速模拟支付流程和测试 API。
