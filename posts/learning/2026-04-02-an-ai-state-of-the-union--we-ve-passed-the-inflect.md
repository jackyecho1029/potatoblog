# 核心分析（金字塔原理）

## 1. 代码成本归零：软件工程的底层逻辑已被重写
- **深度剖析**：2025 年是 Anthropic 和 OpenAI 全面转向代码训练的一年。强化学习+推理模型的组合（O1 → 所有模型跟进）让代码生成质量在 11 月跨过临界点——从"大部分时候基本能用"跃迁到"几乎每次都能按指令完成"。这意味着传统软件工程中"写代码"这个最耗时的环节被压缩了几个数量级，整个开发流程的瓶颈转移到了其他环节：需求定义、架构设计、质量保证、安全审查。
- **实战案例**：Simon 自己在手机上用 Claude Code for Web 完成大部分编程工作，遛狗时就能推进项目。他可以同时启动 4 个 agent 并行工作，上午 11 点就被认知负荷耗尽。他的新年前决心从往年的"做减法"变成了"做加法"——因为 AI 让他能够承担更多项目。

## 2. "暗工厂"模式：不看代码如何保证质量
- **深度剖析**：当代码完全由 AI 编写且人类不再逐行审查时，质量保证的范式必须彻底重构。核心思路是把传统的 QA 团队"模拟化"——用 agent swarm 进行 24 小时不间断测试。关键创新在于：不需要审查过程，只需要验证结果。
- **实战案例**：StrongDM（安全访问管理公司）从 2025 年 8 月开始实践"无人阅读代码"策略。他们用 agent swarm 模拟数千名员工在假的 Slack/Jira/Okta 中发送请求，每天花费约 $10,000 的 token 费用进行持续测试。他们甚至用 coding agent 根据公开 API 文档构建了自己的 Slack 和 Jira 模拟器——一个免费的 Go 二进制文件，避免了真实产品的 rate limit 问题。

## 3. 职业分层：资深和新人是受益者，中层是最大输家
- **深度剖析**：ThoughtWorks 组织的工程 VP 研讨会揭示了一个 U 型曲线——资深工程师的 25 年经验被 AI 放大（能用高级工程语言与 agent 协作），新人受益于 AI 辅助入职（Shopify 和 Cloudflare 各招聘 1000 名实习生，因为上手时间从一个月缩短到一周），但中级工程师两头不靠：既没有足够经验来"放大"，又已经过了新手期的 AI 辅助红利。
- **实战案例**：Simon 指出他能对 agent 用一句专业术语（如"red/green TDD"）就触发一整套工作流，这是 25 年经验结晶。而中级工程师缺乏这种"压缩指令"能力。数据标注公司甚至开始高价收购 2022 年前的手写代码仓库来训练模型——"手工代码"正在成为一种稀缺资源，如同核爆前的沉船金属。

# 芒格格栅：思维模型拆解

- **边际成本趋零效应**：Simon 反复强调"代码现在是廉价的"，这改变了软件工程的所有决策权重。以前避免过度测试是因为维护成本高；现在测试代码的维护成本由 agent 承担，所以他更倾向于写 100+ 测试用例的小型库。这个思维模型适用于所有"某个关键资源成本骤降"的场景——重新审视整个流程中哪些环节的决策逻辑需要重写。

- **能力放大器 vs 能力替代器**：Simon 将 AI 定义为"现有技能和经验的放大器"而非替代品。这与 Clayton Christensen 的颠覆式创新模型形成对比——AI 目前并非从低端替代起步，而是首先赋能高端用户。这意味着拥有深厚领域知识的人反而获得了最大收益，颠覆了"AI 先替代低端工作"的常见假设。

- **偏差正常化（Normalization of Deviance）**：借用挑战者号航天飞机灾难的研究框架，Simon 指出 AI 行业正在重复同样的错误——prompt injection 的安全漏洞每次"没出事"都在增强行业的安全感，直到某天灾难发生。这是一个经典的"黑天鹅"模式。

- **库存思维（Hoarding）**：Simon 的职业策略是不断积累"做过的事情的库存"——193 个小工具、75 个公开研究项目、50 个私人研究项目——这些不是简单的笔记，而是可被 coding agent 检索和组合的"模块化经验"。当新问题出现时，他能组合 2015 年的 Redis 方案和 2017 年的 Node.js 经验来解决当前问题，而 AI 让这种组合的试错成本趋近于零。
