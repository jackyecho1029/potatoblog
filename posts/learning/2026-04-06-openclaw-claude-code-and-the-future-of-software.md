## 📖 詳細內容

### 01｜OpenClaw 的日常使用：不只是一個工具，更像一個人

**核心觀點：** Peter 的 OpenClaw 名叫「Zoe」，他 70-80% 的使用場景是透過 Telegram 做語音對話，而非執行具體任務。Zoe 會給他晨間摘要、提供深度洞察，甚至給他一段三分鐘的「打氣演講」，提醒他小孩很快就長大了，應該把時間留給家人。這種使用體驗比起 ChatGPT 或 Claude 的網頁介面更私人化、更像和一個人對話。

**重要原話：**
> "我覺得它更個人化，比起使用 ChatGPT 或 Claude，它就是一個我可以在床上發訊息的東西。"
>
> （原文："It just feels like more personal than using like Claude or ChatGPT, and it just feels like something I can like text in bed."）

**個人感受：** 我一直把 AI 當作工具使用，但 Peter 的經驗讓我思考：也許最重要的不是它能做多少事，而是它能成為什麼樣的存在。

**延伸思考：** 當 Agent 的介面從「網頁」變成「你隨身的聊天對象」，使用頻率和情感連結都會發生質的變化。

**可參考的行動：** 嘗試在 Telegram 或其他你日常使用的通訊軟體上設定一個 AI Agent，連續使用一週，觀察自己的使用模式和感受變化。

---

### 02｜Claude Code vs Codex：兩種不同的 Coding 節奏

**核心觀點：** Peter 認為 Codex 更精準但速度較慢，像「和一個每次思考三分鐘才回答的人對話」；Claude Code 則更健談、假設更多，但在同步體驗中更愉快。他用 Codex 來認真構建功能，用 Claude Code 來「vibe coding」。兩者都有類似老虎機的「變動獎勵」效果——結果不可預測、時間也不可預測，這讓使用過程有一種上癮感。

**重要原話：**
> "Claude Code 幾乎像一台老虎機，每次出來的東西都不一樣。"
>
> （原文："Claude is almost like a slot machine. It's like it has different things each time."）

**個人感受：** 終於有人講出這個感受了——每次送出 prompt 都不知道會得到什麼，這種不確定性本身就是吸引力的來源。

**延伸思考：** Coding agent 的「娛樂性」可能是它的殺手級特徵之一，這也解釋了為什麼這麼多人沉迷 vibe coding。

**可參考的行動：** 如果你有程式需求，分別用 Claude Code 和 Codex 完成同一個小任務，記錄各自的節奏、準確度和你的使用感受，找到適合自己的場景分工。

---

### 03｜Coding 將吞噬所有知識工作

**核心觀點：** Peter 認為 coding 不再只是寫程式，它正在變成一種通用的問題解決方式。就像 Excel 是世界上最受歡迎的程式語言一樣（Satya Nadella 的觀點），coding agent 將把這個能力放大一千倍。他舉例自己現在寫部落格文章的方式是先讓 AI 生成 80%，然後手動調整最後 20%。一間 AI 原生的 AppGen 公司甚至讓 vibe coder 來取代他們付費的 SaaS 工具。

**重要原話：**
> "我覺得 coding 會吃掉所有知識工作。我們已經在朝那個方向走了。"
>
> （原文："I feel like coding will eat all knowledge work, right? And we're kind of going that direction already."）

**個人感受：** 連寫文章都可以用 coding agent 的方式完成，這讓我重新思考什麼才是不可能被自動化的部分——也許是最初的洞察和最終的判斷。

**延伸思考：** 當「coding」的門檻降到接近零，每個人都變成 builder，競爭的焦點會從「能不能做」轉向「做什麼」和「為誰做」。

**可參考的行動：** 挑一個你日常的重複性知識工作（例如整理數據、寫報告摘要），嘗試用 AI agent 完成前 80%，刻意記錄你省下的時間和你需要手動調整的部分。

---

### 04｜未來公司：小團隊、多 Agent、快慢交替

**核心觀點：** Peter 的預測是未來的公司會盡量保持小規模：一個 2-3 人的產品團隊加上一群 agent，比 10 人的團隊更容易協作。Agent 之間的跨部門協作比人類之間少了情緒因素，更加客觀高效。他還提出了「快慢交替」的工作模式——在局部最優時全速前進，但要到達下一個山頭時需要慢下來「去摸摸草地」。傳統的年度規劃已經不再適用，但你也不能一直全速衝刺。

**重要原話：**
> "我覺得現在很容易用這些 AI 工具同時往十個方向跑。有時候你確實需要慢下來，想清楚你要去哪裡。"
>
> （原文："I think it's very easy now with all these AI tools just going like 10 different directions at once. So sometimes you do have to slow down and try to figure out where you want to go."）

**個人感受：** 「快慢交替」這個概念非常到位——不是一直快，也不是一直想，而是在對的時機切換節奏。

**延伸思考：** 如果一個人帶著一群 agent 就是一家公司，那「創業」的定義可能需要改寫：不再是找團隊和融資，而是找到問題和快速驗證。

**可參考的行動：** 審視你目前的專案或工作，找出可以用 agent 協助的「局部最優」任務，設定一週的 sprint 全速推進，然後安排半天完全不看螢幕的「摸草地」時間來反思方向。

---

### 05｜工作不會消失，只是形態改變

**核心觀點：** a16z 的主持人提出了一個樂觀視角：目前幾乎所有 AI 產品都只能提升生產力而無法 100% 自動化一個職位，最後 10% 仍需要人類。真正的 100% 自動化只出現在客服等少數場景。人類的欲望沒有天花板——今天的奢侈品會變成明天的基本需求，新的工作會不斷產生。Peter 引用了一條推文：「就業市場太差了，我現在只能去追求我的夢想了。」

**重要原話：**
> "也許你失去了工作，但現在你真的可以去做你自己的事情了，而且真的有機會實現它。"
>
> （原文："So maybe you lost your job, but like now you can actually do your own thing and have a shot at actually achieving it."）

**個人感受：** 這種「被迫追求夢想」的反轉視角讓我會心一笑，但也確實反映了當下的現實——門檻降到歷史最低，不做才是最大的風險。

**延伸思考：** 也許 AI 時代最大的公平在於：以前需要團隊、資本、資源才能做的事，現在一個人就能啟動。

**可參考的行動：** 如果你目前有穩定工作，每週撥出 5 小時用 AI agent 嘗試一個你一直想做的 side project；如果你正在找工作，把原本投履歷的時間分一半出來，用 AI 工具建造一個屬於自己的產品或內容。

## 💎 精華收穫

這集對話最值得記住的是幾個判斷：coding agent 正在從程式工具變成通用問題解決工具；未來的公司會小很多，但每個人都能做更多；工作不會消失，但「工作」的定義會徹底改變。Peter 的使用體驗提醒我們，AI 的真正價值可能不只是效率，而是它改變了我們與技術的關係——從使用工具變成與一個存在協作。
